**FAQ Block:**
- **Q:** What is XHTML?
  **A:** XHTML is a stricter version of HTML that combines HTML with XML rules to ensure cleaner, more predictable web code.
- **Q:** Why was XHTML introduced?
  **A:** To improve web page consistency, facilitate better data exchange, and make pages more predictable for search engines and systems.
- **Q:** Did everyone use XHTML properly?
  **A:** No, many developers adopted the header but often served pages as HTML, leading to widespread misuse.
- **Q:** How did XHTML influence later web standards?
  **A:** It paved the way for HTML5, which adopted some of XHTML’s semantics but with more leniency for errors.
