19 марта 1997 года компания Microsoft выпустила Visual Studio 97 — первую версию среды, которая позже станет одним из главных инструментов разработчиков по всему миру.
В тот момент это было заметное изменение. Вместо работы с разными программами разработчикам предложили единый набор, где всё собрано вместе.
До Visual Studio
В середине 1990-х инструменты Microsoft были разрозненными.
Если вы разрабатывали программное обеспечение, скорее всего вы использовали:
Visual Basic для быстрых настольных приложений
Visual C++ для задач, где важна производительность
Visual FoxPro для работы с базами данных
Visual J++ для экспериментов с Java
Все эти инструменты были мощными, но существовали отдельно друг от друга. Разные интерфейсы, разные подходы, разные привычки. Переключаться между ними было неудобно и занимало время.
Идея объединения
Visual Studio 97 стала попыткой упростить этот процесс.
Вместо отдельных инструментов появилась идея единой среды — места, где можно разрабатывать разные части приложения без постоянных переключений.
Сегодня это звучит очевидно, но тогда это была новая идея: одна среда для разных языков.
Что входило в состав
Visual Studio 97 всё ещё представляла собой набор инструментов, но уже в одном продукте:
Visual Basic 5.0
Visual C++ 5.0
Visual FoxPro 5.0
Visual J++ 1.1
MSDN Library с документацией
Инструменты ещё не были полностью объединены, но начали выглядеть и работать более похоже. Уже это заметно упрощало повседневную работу.
Почему это важно
Visual Studio 97 изменила ожидания разработчиков.
Стало казаться естественным, что инструменты должны работать вместе, а не по отдельности. Что не нужно заново привыкать ко всему при смене языка.
Постепенно закрепились вещи, которые сегодня считаются стандартом:
похожие интерфейсы
единые рабочие процессы
встроенная документация
более тесная интеграция отладки
Позже по этому пути пошли и другие среды, такие как Eclipse и IntelliJ IDEA.
Не идеальная система
При этом Visual Studio 97 ещё не была полноценной единой IDE в современном понимании.
Внутри инструменты оставались довольно независимыми. Иногда это всё ещё ощущалось как набор программ, а не единый продукт.
Настоящая цельная среда появится позже — особенно с выходом Visual Studio .NET в 2002 году.
С чего всё началось
Несмотря на ограничения, Visual Studio 97 задала направление.
Сегодня кажется естественным открыть одну программу и писать код на разных языках, отлаживать, читать документацию и управлять проектом в одном месте.
В 1997 году это только начиналось.
